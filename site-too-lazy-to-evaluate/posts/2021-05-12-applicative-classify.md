---
title: 多項式Applicative Functorの一般論（応用編）
---

[以前](2020-11-25-faccat.html) [書いた](2021-01-01-faccat2.html) ような事実を使えば、
ちょうど2つの項からなる*あらゆる*多項式Applicative Functorを、
up to isoで分類することができます。具体的には、次のような`F`に対して、
ありうる`Applicative F`のインスタンスにはどのようなものがあるか、詳しく分類することができます。

```haskell
data F a = F0 (E0 -> a) | F1 (E1 -> a)
    deriving (Functor)

instance Applicative F where
    -- 一般性を失うことなく、次のように仮定できる：
    pure = F0 . const
```

まずは、`M = Ap F ()`がどのようなモノイドになっているのかで分けましょう。`M`はちょうど2つの元を持ちます。

```haskell
-- M はモノイド
type M = Ap F ()

-- M は以下の2つの値だけを持つ
x0, x1 :: M
x0 = mempty = Ap (pure ()) = Ap (F0 (const ()))
x1 = Ap (F1 (const ()))
```

ちょうど2つの元からなる集合上のモノイド`M`は、同型を除いて2種類しかありません。
単位元をx0、単位元でないほうをx1とおくと、`M`の乗積表は次のようになります。

| `<>` | x0 | x1 |
|:----:|:--:|:--:|
|**x0**| x0 | x1 |
|**x1**| x1 | ?  |

表の中で埋まっていないのは、`x1 <> x1`の値を決める1マスだけです。

1. このマスがx0であったなら、`M`は2元の巡回群[C<sub>2</sub>](https://ja.wikipedia.org/wiki/%E5%B7%A1%E5%9B%9E%E7%BE%A4)です。
2. このマスがx1であったなら、`M`は`Bool`上`||`を二項演算として持つモノイド、
   言い換えれば[Any](https://hackage.haskell.org/package/base-4.14.1.0/docs/Data-Monoid.html#t:Any)と同型のモノイドです。

したがって、`M`はこの2種類のモノイドのどちらかに同型です。

### `M`が2元の巡回群 C<sub>2</sub> である場合

`M`が群なので、これまでの議論から`E0`と`E1`の間には全単射があります。同型を除いた違いだけ考えているので、
`E0`と`E1`は同一視して、ひとつの型`E`で考えます。

さらに、この記事での議論から、
適当に同型なApplicativeを取れば`E`上の`M`-作用で決定されるApplicativeだけを考えればよくなります。

`M`が2元の巡回群 C<sub>2</sub> であるとき、`M`-作用とは何でしょうか？

```haskell
act x0 = id :: E -> E
act x1 = f  :: E -> E
f . f
  = act x1 . act x1
  = act (x1 <> x1)
  = act x0
  = id
```

このように書くと、`M`-作用とは対合(二回適用すると必ずもとに戻る関数)だということがわかります。

型`E`が有限個の値だけをもつなら、その個数を n として、同型でない対合は`1+floor(n/2)`種類だけあります。
これは、次のように考えるとわかります。

* `E = {1,2,...,n}`としておく。
* `E`上の対合`f`が与えられたとき、`E`上の全単射`g`をうまく選ぶことで、`f`と同型な対合`g_inv . f . g`が、
  2k個の要素からなるkペアを交換して、残りの要素をそのまま返す関数
  
  ```haskell
  g_inv . f . g :: E -> E
  g_inv . f . g = \i -> case i of
    1 -> 2; 2 -> 1;
    3 -> 4; 4 -> 3;
      :
      :
    (2k-1) -> 2k; 2k -> 2k-1;
    _ -> i
  ```
  
  となるようにできる。対合が動かさない要素の数(n-2k)は同型で変わらないから、
  同型を除いて本質的に異なる対合は、
  ちょうど交換するペアの数`k`のとりうる値{0, 1, ..., floor(n/2)}と同じ個数だけある。

なので、同型を除いて異なるApplicativeも、ちょうど`1 + floor(n/2)`種類だけあります。

型`E`が有限でないなら、少なくとも無限種類の対合がありえますが、その"個数"の厳密な議論はちょっとよくわかりません。

### `M`が`Any`と同型である場合

今度はモノイド`M`が`Any`と同型であるときを考えます。
今度はやや大変なため、表記を簡単にします。`M={0,1}`とし、
単位元を`0`、モノイドの演算子は `+` を使います。つまり、`1 + 1 = 1`です。

圏`MFac`の、全部で8つある射をすべて並べてみます。

```haskell
id_0 = (0,0,0) :: MFac(0,0)

id_1, S, T, U :: MFac(1,1)
id_1 = (0,1,0)
S = (1,1,0) = S∘S
T = (0,1,1) = T∘T
U = (1,1,1) = S∘T = T∘S

S', T', U' :: MFac(1,0)
S' = (1,0,0) = S∘S'
T' = (0,0,1) = T∘T'
U' = (1,0,1) = T∘S'
```

これらが`phi :: MFac(x,y) -> Ex -> Ey`によって写された後の関数は、
8つ全てを考えずとも、次の4つだけで決定されることもわかります。

```haskell
s, t :: E1 -> E1
s', t' :: E1 -> E0
```

ただし、`s = phi (S :: MFac(1,1)) :: E1 -> E1`のように元の文字を小文字化して書くという規約を使いました。
これらの関数たちには、`MFac`から引き継いだ等式

```haskell
s . s = s
t . t = t
s . t = t . s

s . s' = s'
t . t' = t'
s . t' = t . s'
```

が成り立っています。

まず、`s, t :: E1 -> E1`が冪等関数であることから始めます。

一般的に、
有限集合上の冪等関数 \\($f : X \to X$\\) は、同型を除くと、
\\($n = |X|$\\) の分割と同じ個数だけあります。\\($n$\\) の[分割](wikipedia)とは、
合計が\\($n$\\)になる \\($k\ge 1$\\) 個以上の正整数の組で、大きい順にソートされているようなもののことです。

以下の図のように、$X$の各要素を頂点とし有向辺$(x,f(x))$をもつようなグラフは、
$f$が冪等関数であったならばいくつかの星型の（弱）連結成分からなることを見てください。
"同型を除く"ことが、頂点のラベルを忘れることに相当することを考えると、
同型を除いた冪等関数が、$n$の分割と一対一に対応することがわかります。
